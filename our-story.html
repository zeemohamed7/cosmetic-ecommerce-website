Our Story

Celeste began with a simple idea: create clean, elegant beauty products shaped by science and designed for everyone.

Project Vision

Our vision is to build a cosmetics experience that blends research-backed formulas, ethical craftsmanship, and timeless design.

Reflection

Throughout this project, we collaboratively designed and developed Celeste, a multi-page cosmetics website that applies core Web Fundamentals concepts such as semantic HTML, CSS styling, JavaScript interactivity, responsive Bootstrap layouts, and consistent branding. Our work included building the Homepage, Science page, Our Story page, Making Of page, and a fully functional Registration Form with dynamic validation and user-friendly feedback.

Our project began with creating a unified brand identity. We built a color system using CSS variables (:root), which allowed us to apply bronze, cream, peach, pink, and brown consistently throughout the website. We also chose Google Fonts (Roboto) for a clean and modern style. Hero images and product images were created using AI tools to match the soft, elegant Celeste theme. Structurally, we applied semantic HTML5 elements such as section, header, main, and footer for clarity and long-term maintainability.

We faced several challenges during development. One major difficulty was working with JavaScript before it was fully covered in the syllabus. Implementing the registration form required regex-based email validation, character length checks, and dynamic error messages. To overcome this, we relied on documentation, online examples, and internal group support. Combining JavaScript validation with Bootstrap’s .was-validated, .invalid-feedback, and .is-valid classes required experimentation and debugging until the form behaved correctly.

Another challenge involved overriding Bootstrap’s default styling. Some Bootstrap utilities were conflicting with our custom design system, especially with form controls and spacing. We overcame this by using targeted selectors (such as #registerForm .form-control), increasing CSS specificity, and reorganizing our custom rules. We also used the browser inspector to fix layout issues, adjust the box model, correct spacing problems, and ensure responsiveness on smaller screens.

In terms of group work, this project taught us the value of clear communication and regular updates. Sharing ideas early prevented misunderstandings and ensured everyone stayed aligned with the brand style and functionality. We divided responsibilities based on strengths—for example, some focusing more on JavaScript logic while others handled layout, page content, and documentation. Using GitHub helped us collaborate smoothly, commit changes safely, and make sure the entire group always worked on the latest version of the project.

Technically, this project strengthened our understanding of web development. We learned how to structure webpages properly using semantic HTML, create scalable and maintainable styling using CSS variables, and debug issues related to specificity and the cascade. We became more comfortable with Bootstrap, especially responsive grids, cards, navbars, and form components. In JavaScript, we learned how to use addEventListener(), regex patterns for validation, DOM manipulation, and localStorage to personalize the navigation bar by showing the user's name.

We also learned how to create interactive UI sections such as ingredient tabs using Bootstrap pills, thumbnail click effects, and dynamic form messages. Debugging became easier as we practiced reading console messages, checking for missing brackets, and inspecting CSS layers. Overall, this project helped us transform theoretical lessons into a functioning, polished website and strengthened both our technical abilities and our confidence in working collaboratively on real web projects.
